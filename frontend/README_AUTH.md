# Режим разработки для авторизации

Если сервис отправки кодов (MSG OVRX) недоступен, фронтенд автоматически переходит в **режим разработки**.

## Как это работает:

1. При попытке отправить код, если сервис недоступен, на экране появится сгенерированный код (например: `123456`)
2. Этот код нужно добавить в базу данных вручную

## Добавление кода в базу данных:

### Вариант 1: Через скрипт (рекомендуется)

```bash
python scripts/add_test_code.py <email_or_phone> <code>
```

Пример:
```bash
python scripts/add_test_code.py test@example.com 123456
python scripts/add_test_code.py +79991234567 123456
```

### Вариант 2: Через API документацию

1. Открой http://localhost:8000/docs
2. Найдите эндпоинт `POST /auth/send-code`
3. Отправь запрос с email/телефоном
4. Если появится ошибка, используй код из сообщения на экране
5. Добавь код вручную через SQL или используй скрипт выше

### Вариант 3: Прямой SQL запрос

```sql
INSERT INTO auth_codes (identifier, code, created_at, is_used)
VALUES ('test@example.com', '123456', datetime('now'), 0);
```

## После добавления кода:

1. Вернись на страницу авторизации
2. Введи email/телефон, который использовал при добавлении кода
3. Введи код, который добавил в базу
4. Нажми "Подтвердить код"

## Примечание:

В режиме разработки код показывается на экране для удобства тестирования. В продакшене этого не будет.
