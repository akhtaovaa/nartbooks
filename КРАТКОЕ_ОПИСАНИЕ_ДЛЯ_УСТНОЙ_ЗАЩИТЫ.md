# Краткое описание проекта NartBooks для устной защиты

## Введение (1-2 минуты)

Добрый день! Мой проект называется **NartBooks** — это веб-приложение для книжного клуба, представляющее собой лендинг с системой управления книгами месяца, авторизацией пользователей и административной панелью.

**Актуальность проекта:** В современном мире книжные клубы нуждаются в удобной платформе для привлечения участников, управления контентом и организации встреч. Мой проект решает эти задачи.

---

## Цели и задачи (1 минута)

**Основные цели:**
1. Создать информативный лендинг для привлечения новых участников
2. Реализовать систему управления книгами месяца
3. Обеспечить авторизацию пользователей через систему кодов верификации
4. Предоставить административную панель для управления контентом

**Задачи:**
- Разработать RESTful API на FastAPI
- Создать современный frontend на HTML/CSS/JavaScript
- Реализовать систему авторизации через JWT токены
- Обеспечить безопасность и разделение прав доступа

---

## Функционал проекта (2-3 минуты)

### Для обычных пользователей:
- **Главная страница (лендинг):** просмотр информации о клубе, текущей книги месяца, списка всех книг с поиском
- **Авторизация:** вход через email/телефон с кодом верификации, автоматическое создание аккаунта
- **Личный кабинет:** просмотр и редактирование профиля, управление избранным, просмотр записанных встреч
- **Работа с книгами:** просмотр детальной информации, добавление в избранное, оставление отзывов, запись на встречи

### Для администраторов:
- **Управление книгами:** добавление, редактирование, удаление книг месяца, установка текущей книги, просмотр статистики
- **Управление пользователями:** просмотр списка пользователей, изменение ролей, просмотр статистики каждого пользователя

---

## Технологии и архитектура (2-3 минуты)

### Backend:
- **Python 3.9+** с фреймворком **FastAPI** для создания RESTful API
- **SQLAlchemy** как ORM для работы с базой данных
- **SQLite** (можно легко заменить на PostgreSQL)
- **JWT токены** для авторизации
- **Pydantic** для валидации данных

**Архитектура:** Модульная структура с разделением на роутеры (auth, books, users, favorites, meetings), модели данных, схемы валидации и зависимости.

### Frontend:
- **HTML5, CSS3, JavaScript (ES6+)**
- Модульная структура: отдельные модули для API, авторизации, админ-панели
- **Fetch API** для взаимодействия с backend
- Адаптивный дизайн

**Взаимодействие:** Frontend отправляет HTTP запросы к API, получает JSON ответы, авторизованные запросы включают JWT токен в заголовке Authorization.

---

## Основные API эндпоинты (1-2 минуты)

### Авторизация:
- `POST /auth/send-code` — отправка кода верификации
- `POST /auth/verify-code` — верификация кода и получение токена

### Книги:
- `GET /books/current` — текущая книга месяца
- `GET /books` — список книг с пагинацией и поиском
- `POST /books`, `PUT /books/{id}`, `DELETE /books/{id}` — управление книгами (только админы)

### Пользователи:
- `GET /me` — информация о текущем пользователе
- `GET /users` — список пользователей (только админы)
- `PUT /users/{id}/role` — изменение роли (только админы)

### Дополнительно:
- Избранное, отзывы, записи на встречи

**Документация API:** Автоматически генерируется через Swagger UI по адресу `/docs`

## Система безопасности (1 минута)

1. **JWT токены:**
   - Ограниченное время жизни (24 часа)
   - Хранение в базе данных для возможности отзыва
   - Алгоритм HS256 для подписи

2. **Защита административных эндпоинтов:**
   - Проверка роли через зависимость `require_admin_role`
   - Автоматическое присвоение роли admin только для определенных email

3. **Валидация данных:**
   - Pydantic схемы для всех входных данных
   - Защита от SQL-инъекций через ORM
   - Экранирование HTML на frontend

## Ответы на возможные вопросы

### Вопрос: Почему выбрали FastAPI?
**Ответ:** FastAPI — современный, быстрый фреймворк с автоматической генерацией документации, встроенной валидацией данных через Pydantic и поддержкой асинхронности.

### Вопрос: Как работает авторизация?
**Ответ:** Пользователь вводит email, получает код верификации (в режиме разработки код возвращается в ответе), вводит код и получает JWT токен, который используется для авторизованных запросов.

### Вопрос: Как обеспечивается безопасность?
**Ответ:** JWT токены с ограниченным временем жизни, проверка ролей на уровне зависимостей FastAPI, валидация всех входных данных, защита от SQL-инъекций через ORM.

### Вопрос: Можно ли расширить функционал?
**Ответ:** Да, модульная архитектура позволяет легко добавлять новые функции. Например, можно добавить новые роутеры, расширить модели данных, добавить новые страницы на frontend.

### Вопрос: Как развернуть в production?
**Ответ:** Заменить SQLite на PostgreSQL, настроить HTTPS, использовать переменные окружения для всех секретов, настроить reverse proxy (nginx), добавить логирование и мониторинг.
